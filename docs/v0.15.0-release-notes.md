# v0.15.0 — Minimal Demo UI

## Summary

This release adds a minimal React + Vite + TypeScript demo UI for the `hotelbooking` platform.

The UI is designed as a thin portfolio/demo client for the backend architecture. It demonstrates the main distributed booking flow without moving business logic to the frontend.

## Added

### Demo UI application

New application:

```text
apps/demo-ui
```

Features:

- hotel catalog page;
- hotel details and room type selection;
- room availability check;
- booking saga creation;
- current user bookings page;
- booking audit timeline;
- booking cancellation action;
- inventory admin page for local demo data;
- Google authentication mode;
- demo authentication mode for local development.

### Google auth mode

The UI can sign in with Google and send Google ID token to `booking-service`:

```http
Authorization: Bearer <google-id-token>
```

Recommended backend profile:

```text
dev-jwt
```

### Demo auth mode

For local runs without Google, the UI can be switched to:

```env
VITE_AUTH_MODE=demo
```

Recommended backend profile:

```text
dev
```

### Booking-service profile group

Added JWT-oriented local development profile group:

```text
dev-jwt
```

This mirrors `dev`, but uses `security-jwt` instead of `security-dev`.

### Observability logging profile support

The shared Logback configuration should treat `dev-jwt` as a local/development profile so colored logs remain enabled when JWT security is active.

## Demo flow

```text
Inventory Admin
  -> create hotel
  -> add room type
  -> initialize availability

Hotels
  -> check availability
  -> start booking saga

My bookings
  -> inspect status
  -> open timeline
  -> cancel booking
```

## Notes

The frontend is intentionally simple. The architectural focus remains on backend design:

- DDD module boundaries;
- application/domain separation;
- saga orchestration;
- outbox publishing;
- Kafka integration;
- gRPC inventory integration;
- JWT/dev security modes;
- audit and observability.
